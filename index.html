🌊 너비 우선 탐색 (BFS)

루트 노드에서 시작해 같은 레벨(깊이)의 노드를 모두 방문한 후 다음 레벨로 이동합니다. 큐(Queue) 자료구조를 사용합니다.

예시: 최단 경로 찾기, 지하철 노선 환승 최소화
특징: 항상 최단 경로 보장, 메모리 사용량 많음
⬇️ 깊이 우선 탐색 (DFS)

루트 노드에서 시작해 한 경로를 끝까지 탐색한 후 되돌아와 다른 경로를 탐색합니다. 스택(Stack)/재귀를 사용합니다.

예시: 미로 탈출, N-Queen 백트래킹
특징: 메모리 효율적, 최단 경로 미보장

🧠 휴리스틱 탐색

평가 함수를 활용하여 언덕 등반 · 최고 우선 · A* 알고리즘을 이해하고 직접 탐색 경로를 완성해봅니다.

🧠 휴리스틱 탐색 (Heuristic Search)

휴리스틱(Heuristic)이란 현재 상태에서 목표까지의 예상 비용을 추정하는 함수입니다. 이 정보를 활용해 탐색 공간을 줄이고 더 빠르게 해답을 찾습니다.

⛰️

언덕 등반 탐색

현재 상태에서 가장 좋아 보이는 이웃 상태로 이동. 빠르지만 지역 최적해에 빠질 수 있음.

예시: GPS 경로에서 가장 가까운 방향으로만 이동
⭐

최고 우선 탐색

휴리스틱 값이 가장 좋은 노드를 우선 탐색. 우선순위 큐를 사용하여 효율적으로 탐색.

예시: 목적지까지 직선 거리가 가장 짧은 경로 우선 탐색
🎯

A* 알고리즘

f(n) = g(n) + h(n)
실제 비용 g(n)과 휴리스틱 h(n)을 합산. 최적 경로 보장.

예시: 게임 캐릭터 이동, 내비게이션 최단 경로
⛰️ 언덕 등반 탐색 (Hill Climbing)
⛰️ 언덕 등반 탐색이란?

현재 상태에서 이동 가능한 노드 중 평가 함숫값이 가장 큰 노드로 이동하여 경로를 탐색하는 방법입니다.
등산할 때 항상 더 높은 방향으로만 이동하는 것에 비유할 수 있습니다.

📌 평가 함수 (Heuristic)
현재 노드에서 목표 상태와 비교했을 때 제 위치에 놓여있는 타일의 수
→ 값이 클수록 목표에 가까운 상태
🔄 탐색 순서
1 현재 노드의 자식 노드 평가 함숫값 계산
2 가장 높은 값의 노드로 이동
3 이동 후 값이 현재보다 크지 않으면 탐색 중단
✓ 목표 상태 도달 시 탐색 종료
⚠️ 한계: 지역 최적해(Local Maximum)에 빠질 수 있어 항상 최적 경로를 보장하지 않습니다.

가장 효율적으로 목표 상태를 찾는 것을 목적으로 하는 방법.
목표 상태와 최단 경로를 찾는 상황을 고려한 두 가지 평가 함수를 활용합니다.

평가 함수 (Evaluation Function)
f(n) = g(n) + h(n)
f(n)이 작을수록 목표에 가까운 상태

♛ N-Queen 문제

N×N 체스판에 N개의 퀸을 서로 공격하지 않도록 배치하는 문제입니다. 백트래킹 탐색을 통해 해결합니다.

♛ N-Queen 문제란?

N-Queen 문제

체스에서 퀸은 한 번 움직일 때 상하, 좌우, 대각선 모든 방향으로 이동할 수 있는 말입니다. 4-Queen은 체스 퀸 4개가 서로를 공격하지 않도록 4×4 크기의 체스판에 배치하는 게임입니다. 즉, 4×4 체스판의 각 열마다 퀸을 하나씩 배치하되 모든 퀸이 서로를 공격하지 않도록 배치하는 게임입니다.

📊 초기 상태, 목표 상태, 퀸의 이동

① 초기 상태: 4×4 체스판에 어떠한 말도 올려놓지 않은 상태
② 목표 상태: 4×4 체스판의 각 열마다 퀸을 하나씩 배치하되 모든 퀸이 서로를 공격할 수 없는 상태 (검은색이 퀸의 위치)
③ 퀸의 이동 가능한 방향: 퀸은 한 번 움직일 때 체스판의 상하, 좌우, 대각선의 모든 방향으로 이동할 수 있는 말입니다.

🌳 탐색 트리 (Search Tree)

📌 학습 방법: 레벨 0~2는 미리 채워져 있습니다.
레벨 3~4 노드(🟡 노란 테두리 칸)에서 퀸을 놓을 위치를 클릭하여 정답을 맞춰보세요.
정답을 맞추면 퀸이 서로 공격하지 않는 이유를 선으로 시뮬레이션해 드립니다.
